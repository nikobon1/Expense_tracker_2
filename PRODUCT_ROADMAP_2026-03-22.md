# Product Roadmap (2026-03-22)

Ниже зафиксирован практический roadmap на основе handoff-файлов, текущего состояния `vercel-app` и доработок dashboard, сделанных к 22 марта 2026 года.

## 1. Уже готово

- Вход в web-приложение через Google OAuth.
- Защищённый dev/staging login с обязательным `DEV_LOGIN_PASSWORD`.
- Загрузка фото чека через web UI.
- AI-распознавание чека и сохранение расходов в базу.
- Quick Add / ручное добавление покупки без фото.
- Полноценное редактирование и удаление чеков.
- Работа с категориями, включая пользовательские категории.
- Dashboard с фильтрами по периоду, магазину и категориям.
- Аналитика по категориям и по дням.
- Сравнение выбранного периода с предыдущим сопоставимым диапазоном.
- Экспорт расходов в Excel.
- Telegram-бот с приёмом фото чеков.
- Telegram manual flow для добавления расходов без фото.
- Общая Postgres-база для web и Telegram.
- Atomic receipt writes и явные миграции через `npm run db:migrate`.
- Новый mobile-first dashboard в стиле Stitch с русским интерфейсом.

## 2. В доводке

- Финальная полировка dashboard UX после переноса из Stitch.
- Дочистка русской локализации интерфейса за пределами уже обновлённого dashboard.
- Выравнивание типографики, контраста и консистентности отдельных блоков мобильного dashboard.
- Упрощение документации, чтобы актуальный продуктовый контур было проще считывать без handoff-файлов.

## 3. Следующие инженерные шаги

- Добавить общие схемы валидации API payloads и корректные `4xx`-ответы.
- Разбить dashboard на более мелкие компоненты и вынести дублирующиеся клиентские утилиты.
- Свести legacy Streamlit и Next.js бизнес-правила в один канонический domain layer.
- Дочистить предупреждение Next.js про multiple lockfiles / inferred Turbopack root.
- При необходимости сделать более широкий проход по старым docs/handoff-файлам на предмет кодировки и консистентности.

## 4. Что не считается roadmap-фичами

Ниже не включено как отдельные продуктовые пункты:

- временные локальные dev-only настройки
- разовые MCP / DevTools проверки
- служебные deploy notes
- commit hashes и операционные команды
